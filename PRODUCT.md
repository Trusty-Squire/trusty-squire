# Product

## Register

Brand

## Users

Trusty Squire is for developers who work primarily through coding agents such as Claude Code, Codex, Cursor, and Goose. They want their agent to finish setup work instead of stopping at a login screen, but they do not want credentials copied into chat, source code, or `.env` files.

## Product Purpose

Trusty Squire signs up / in to websites for you so you don't have to.

It gives coding agents a safe way to create accounts, sign in, configure services, and save the resulting API keys or credentials directly to a vault. Success means a developer understands that job in one sentence, sees a concrete use case that matches their work, and can install Trusty Squire without learning internal architecture first.

## Brand Personality

Calm, exact, capable, and serious. Trusty Squire should feel like dependable infrastructure for people who live in a terminal, with plain language and visible proof instead of hype.

## Anti-references

- Abstract pitch-deck language such as "credentials broker" without an immediate plain-English explanation
- Generic SaaS layouts, interchangeable feature cards, and vague productivity claims
- Consumer password-manager positioning
- Magical or autonomous claims that hide what the agent actually does
- Fear-driven security copy or unsupported promises
- Decorative complexity that makes installation and use cases harder to find

## Design Principles

1. Lead with the literal job. Say what Trusty Squire does before explaining how it works.
2. Show real requests early. Concrete asks such as "sign me up for Resend" are more useful than capability labels.
3. Keep the interface calm and technical. Strong typography, ruled structure, and terminal evidence should carry the brand.
4. Explain safety in plain language. Credentials go to the vault, not into chat or source code.
5. Make every discovery surface consistent. The website, GitHub, npm, directories, and search metadata should describe the same product.

## Accessibility and Inclusion

Use semantic structure, complete keyboard access, visible focus states, sufficient color contrast, and reduced-motion support. Never rely on color alone to communicate state. Keep copy direct and readable for developers who are new to agents as well as experienced users.
